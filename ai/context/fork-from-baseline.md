## Operator checklist — TODO (no script can do these)

These provision real cloud resources and are intentionally **not** done by the
fork. Track them here. `workersDevSubdomain` was left as the placeholder
`your-workers-subdomain` — set the real one (re-run `rebrand.mjs` with it, or
edit the app-config seams) before any deploy.

- [ ] **GitHub Actions secrets**: `CLOUDFLARE_ACCOUNT_ID`,
      `CLOUDFLARE_API_TOKEN` (Workers+KV+Hyperdrive+DNS scopes), `SUPABASE_API_KEY`.
- [ ] **AWS** (via the org's `aws-admin` repo): GitHub-OIDC roles
      `<env>-github-<org>-lumen-{plan,production-deploy}` per environment, plus
      Secrets Manager write scope `<org>/lumen/*`. State buckets are shared org infra.
- [ ] **Supabase**: org access for the management token. `lumen-stage` /
      `lumen-prod` projects are created by the `supabase` Terraform component on first apply.
- [ ] **Cloudflare**: account + real `workersDevSubdomain`, zone for `lumen.app`,
      DNS delegation; `stage.lumen.app` / `prod.lumen.app` attach via `cloudflare-domain`.
- [ ] **OAuth apps** (GitHub + Google, per env): set client IDs in
      `apps/identity-worker/wrangler.template.jsonc` vars; load secrets with `wrangler secret put`.
- [ ] **GitHub Apps** for the integrations cluster (per-env registration; see
      `specs/epics/saas-integrations/`).
- [ ] **Billing**: Polar (or Stripe) products and the env secrets the billing-worker expects.

## First-boot expectations (from FORKING.md §5)

- PR lanes plan Terraform only; the `cloudflare-hyperdrive` plan is red on PRs
  until the first `main` apply writes the Supabase credentials to Secrets Manager.
- Re-run the **full** workflow, never "re-run failed jobs" (orun state keys on `<run>-<attempt>`).
- Keep PRs to a few components (fleet-wide PRs fan out 30–70 CI jobs).
